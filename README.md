# Port Scan Detection using Wireshark & Nmap

## Overview

This project demonstrates how to detect and analyze TCP port scanning activity using Wireshark while generating traffic with Nmap.

The objective is to understand how reconnaissance traffic appears on the network and how a SOC analyst can identify scanning behavior using packet analysis.

## Wireshark Analysis

The following statistics were analyzed:

- Capture File Properties
- Protocol Hierarchy
- Conversations
- Endpoints

Filters used:

- tcp
- tcp.flags.syn == 1 && tcp.flags.ack == 0
- tcp.flags.syn == 1 && tcp.flags.ack == 1
- tcp.flags.reset == 1
- tcp.port == 22
- tcp.port == 80

---

## Key Findings

- More than 11,000 packets captured.
- TCP represented 98.3% of all traffic.
- More than 5,000 SYN packets detected.
- Over 5,000 RST packets generated by closed ports.
- Only ports 22 and 80 responded with SYN-ACK.
- Traffic pattern clearly indicates automated Nmap scanning.
